# Arda

A landing page for **Arda** — original songs in J.R.R. Tolkien's **Sindarin**,
written in the **Tengwar** script with English translations. One atmospheric
scrolling page whose centrepiece is a synced-lyrics "stage" that lights each
line as the song reaches it. Three mood themes (Twilight Gold · Moonlit Silver ·
Ember / Mourning).

Standalone, vanilla HTML/CSS/JS — **no framework and no build step to run.**

## Run it

Serve the folder over http (so synced `.lrc` files can load):

```bash
python3 -m http.server 8777
# open http://localhost:8777
```

## Structure

```
index.html            markup shell (hero, stage, archive, colophon)
assets/app.css        three themes + all components
assets/app.js         player, LRC sync, themes, archive, hero motes
assets/fonts/         Tengwar Annatar (glaem-unicode .ttf)
data/songs.js         canonical lyrics: Sindarin + English per line  ← edit here
data/tengwar.js       generated Tengwar per line (do not hand-edit)
audio/                web MP3s (transcoded from songs/)
lyrics/               drop synced .lrc files here (see lyrics/README.md)
songs/                WAV masters (git-ignored)
scripts/              one-time asset generators
docs/superpowers/specs/  the design spec
```

## Regenerating assets

The two scripts only need to run when the source material changes; the site
itself needs neither.

**Audio** — transcode the WAV masters to web MP3:

```bash
bash scripts/transcode-audio.sh
```

**Tengwar** — after editing `data/songs.js`, regenerate the script:

```bash
node scripts/transcribe-tengwar.mjs
```

This drives [Glǽmscribe](https://github.com/BenTalagan/glaemscribe) in the
*Sindarin — Tengwar (general use)* mode and writes `data/tengwar.js`. The engine
is AGPL and is used **only as a build tool** (vendored under `.tooling/`,
git-ignored); none of it ships with the site. Set `GLAEMSCRIBE_JS` to point at a
different engine checkout. Eyeball new lines against
[Tecendil](https://www.tecendil.com) (Sindarin mode) — mismatches are a one-line
fix in `data/songs.js`, then re-run.

## Deploy

Same static pattern as `lyrics-sync`: commit the files, add a `CNAME` +
`.nojekyll`, and a GitHub Pages workflow. Everything except `songs/*.wav` and
`.tooling/` is shippable.
